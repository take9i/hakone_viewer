# 最小 MVP の開発工数 — 検討

本資料では、本アプリの**最小 MVP（Minimum Viable Product）に含める機能と、その開発工数**の目安をまとめる。既存の検討資料（[README](README.md)、[db-ui-construction-options](db-ui-construction-options.md)、[cad-search-and-delivery-assessment](cad-search-and-delivery-assessment.md) 等）で示した Phase 1・MVP 推奨構成を前提とする。

---

## 1. 最小 MVP の範囲

### 1.1 含めるもの


| 領域              | 内容                                                                                                                                                                                                                    |
| --------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Cesium ビューア** | 3D 地形・建物オープンデータの表示、日時に応じた太陽の影の描画。**選択した 1 件の CAD**（3D Tiles または glTF の URL／ion アセット ID）を地形上に重畳表示。日時変更 UI（影の確認用）。※既存プロジェクトに地形・建物タイル・影は実装済み。                                                                            |
| **データ・登録**      | 物件（CAD）とメタデータの登録。**Airtable** を UI とし、1 レコード＝1 物件。小さい CAD は Airtable の添付、大きい CAD は **S3 にアップロードした URL** を Airtable に格納。MVP では登録は **Airtable 上で手動** でも可。必要なら「自前フォーム → S3 アップロード → Airtable に URL を書き込む」簡易 API を 1 本用意。 |
| **検索・一覧**       | **条件検索**（Airtable のフィルタまたは簡易フォーム）のみ。自然言語検索は **含めない**。検索結果を**表形式で一覧表示**し、行クリックでそのレコードに紐づく CAD（3D Tiles／glTF の URL または ion アセット ID）を Cesium ビューアに渡して表示する。                                                               |
| **認証**          | **最小限**。オプション A: 認証なし（社内・デモ用）。オプション B: **Supabase Auth** でログインのみ。プラン・課金は MVP では含めない。                                                                                                                                  |
| **配信**          | フロントを **Vercel** 等でホスト。Airtable は既存の Base を利用。S3 は既存バケットまたは新規 1 つ。                                                                                                                                                    |


**登録する CAD の形式と配信元（MVP の想定）**

- **DXF→glTF 等の変換**: MVP では**含めない**。登録するのはあらかじめ 3D Tiles または glTF に変換済みのデータとする（変換は手動・別ツールで行う想定）。  
- **配信元は次のいずれか（または併用）**:
  - **(A) 自前で S3 等に保存**: 変換済み glTF／3D Tiles を S3（または Supabase Storage 等）にアップロードし、得られた **URL** を Airtable に格納。ビューアはその URL を読み込んで表示する。
  - **(B) Cesium ion に登録**: glTF 等を ion にアップロードして 3D Tiles 化し、返ってきた **アセット ID** を Airtable に格納。ビューアは `Cesium.IonResource.fromAssetId(id)` で表示する。

MVP の「CAD 登録まわり」の工数は、(A) の場合は S3 アップロード API 等、(B) の場合は ion REST API でのアップロード〜アセット ID 取得までを想定する。どちらか一方でも、両方対応でもよい。

**評価を考える場合の補足**: MVP として評価してもらうには、DXF をそのまま登録して表示できると説得力が増すため、**DXF→glTF 変換**を MVP に含める選択肢もある。ただし MVP に含めなくてもよい。また **DXF は緯度・経度を持たない想定**のため、登録したモデルを地形上で正しい位置・向きに置く **位置合わせ UI**（ドラッグ・回転で合わせ、緯度経度・高度・ヘディング等を DB に保存する機能）が必要になる。いずれも MVP スコープ外とする場合は、工数は本資料の**補足**に記載した目安を参照できる。

### 1.2 含めないもの（MVP ではスコープ外）

- **影の制約の根本対応** — Cesium のシャドウマップ仕様に起因する問題（太陽を背にすると影が消える／欠ける、視野外の地形が落とす影が描画されない）は、MVP では現状の挙動のままとする。解決できないと実利用が難しい可能性が高く、解決できれば差別化要因になるため MVP 後の優先検討とする。詳細は [cesium-shadow-issues.md](cesium-shadow-issues.md) 参照。スコープ外項目の概算工数は本資料の**補足**に記載。
- 自然言語検索（LLM による Text-to-SQL 等）
- 資材搬入可否（道路幅・電柱の表示・評価）
- 課金・プラン制限（Stripe 等）
- kintone / Salesforce 連携
- 組織単位の権限・マルチテナント
- **CAD の自動変換（DXF→glTF 等）**。MVP では変換パイプラインは作らず、事前に 3D Tiles または glTF に変換済みのデータを、上記のとおり **自前 S3 の URL** または **ion アセット ID** として登録する想定。評価上必要なら MVP に含める選択肢あり。工数は**補足**に記載。
- **位置合わせ UI**（緯度経度を持たない DXF 等のモデルを、地形上でドラッグ・回転して位置・向きを合わせ、緯度経度・高度・ヘディング等を DB に保存する機能）。MVP では含めない想定。工数は**補足**に記載。

## 2. 工数見積もりの前提

- **単位**: 人日（1 人日 ＝ 1 人が 1 日フルで作業する想定）。
- **前提**: 既存リポジトリの Cesium ビューア（地形・建物 3D Tiles・影・clock）を流用する。新規は「選択 CAD の読み込み」「検索・一覧 UI」「必要に応じて認証・アップロード API」。
- **幅**: 最小（認証なし・手動登録のみ・UI は簡素）〜 標準（認証あり・S3 アップロード API あり・一覧 UI を整える）で範囲を記載する。

## 4. 合計工数の目安


| パターン   | 含めるもの                                                | 合計（人日）        |
| ------ | ---------------------------------------------------- | ------------- |
| **最小** | 認証なし、登録は Airtable 手動のみ、UI は簡素                        | **約 11 〜 19** |
| **標準** | Supabase Auth ログイン、S3 アップロード API ＋ 登録フォーム、一覧 UI を整える | **約 14 〜 26** |


- **最小**: 1 + 2 + 3（最小） + 4（0） + 5（0） + 6 + 7 + 8 → 3+1+3+0+0+1+1+2 ＝ 11 〜 5+2+5+0+0+2+2+3 ＝ 19。
- **標準**: 上記に 4（最大 3）、5（最大 2）を足す → 14 〜 26。

**おおよその期間の目安**（1 人でフル稼働の場合）: **約 2 〜 4 週間**（最小 〜 標準）。

## 5. リスク・バッファ

- **CAD 形式・変換**: MVP では「事前に 3D Tiles／glTF 化済みの URL を登録」とすれば変換パイプラインの工数は不要。ion を利用する場合は、登録時に ion REST API でアップロード〜タイル化する処理を追加する場合、**+2 〜 4 人日**程度を見ておくとよい。
- **Airtable API 制限**: レート制限や 1 Base あたりのレコード数。件数が少ないうちは問題にならない想定。
- **影の制約**: 上記 1.2 のとおり。根本対応は MVP スコープ外。実利用・差別化の観点から MVP 後の優先検討とする。

## 6. 補足: MVP スコープ外項目の概算工数

以下は 1.2 で「含めないもの」とした機能を、MVP 後に実装する場合の**概算工数**（人日）である。前提・範囲によって変動するため、参考用の目安とする。

| 項目 | 概要 | 概算工数（人日） |
|------|------|------------------|
| **影の制約の根本対応** | シャドウマップのカリングを光視錐ベースに変更する Cesium 側の対応、またはシャドウマップの保持・再利用。 | **10 〜 25** |
| **自然言語検索** | LLM で自然言語→検索条件（Airtable filterByFormula 等）への変換 API、プロンプト設計、エラーハンドリング、一覧 UI との接続。 | **3 〜 8** |
| **資材搬入可否** | 道路幅員・電柱データの取得・取り込み、Cesium/地図上への表示、必要に応じて NAVITIME 等 API 連携と簡易評価ロジック。 | **8 〜 20** |
| **課金・プラン制限** | Stripe 組み込み（Checkout／サブスク）、Webhook、プランと機能の紐付け、Customer Portal 等。 | **5 〜 12** |
| **kintone / Salesforce 連携** | 各プラットフォームの API・認証・iframe 埋め込み、ストア規定やセキュリティレビューへの対応。1 つあたり。 | **5 〜 15** |
| **組織単位の権限・マルチテナント** | 組織・メンバー管理、データの組織スコープ、プランと組織の紐付け、UI。 | **8 〜 20** |
| **CAD の自動変換（DXF→glTF 等）** | DXF 等のアップロード→変換（既存ツール連携 or 自前パイプライン）→glTF／3D Tiles 化→ion または S3 保存、ジョブ管理。MVP として評価を厚くする場合は検討対象。 | **5 〜 15** |
| **位置合わせ UI** | 緯度経度を持たない DXF 等のモデルを、Cesium 上でドラッグ・回転して地形上の位置・向きを合わせる UI。合わせた緯度経度・高度・ヘディング（等）を Airtable 等に保存し、次回表示時に反映。 | **3 〜 8** |

※ 影の根本対応は Cesium 本体の変更を伴う場合があり工数幅が大きい。シャドウマップの保持・再利用のみであれば下振れ側に近い。
