# 3D 地形・地図タイル・建物オープンデータの入手と配信 — 方法の整理

本資料では、Cesium で表示する **3D 地形**・**地形に貼る地図（影像）**・**建物オープンデータ**について、**入手方法**と**配信方法**の選択肢をざっとまとめる。詳細な比較・選定は別途行う想定である。

---

## 1. 3D 地形（Terrain）

### 1.1 入手方法


| 方法                                       | 概要                                                                                      | 備考                                                             |
| ---------------------------------------- | --------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| **Cesium ion（Japan Regional Terrain 等）** | 国土地理院の基盤地図等を元にした quantized-mesh 地形が ion 上で配信されている。日本域は高解像度（1m〜10m）。無料アカウントでストリーミング利用可能。 | 本プロジェクトでは既に `C.IonResource.fromAssetId(770371)` で利用。申請不要・即利用可。 |
| **国土地理院 基盤地図情報・数値標高モデル**                 | 基盤地図情報ダウンロードサービスで数値標高モデル（DEM）を無償ダウンロード可能。座標系は JGD2024 等。                                | タイル形式ではないため、自前で quantized-mesh 等に変換する必要がある。                    |
| **シームレス標高タイル（産総研等）**                     | 標高をウェブ地図のタイル形式で配信。統合 DEM・陸域統合 DEM 等。                                                    | Cesium の terrain として使うには quantized-mesh への変換パイプラインが必要。         |
| **PLATEAU 関連**                           | 3D 都市モデルに含まれる地形・土地起伏。CityGML 等で提供されることがある。                                              | 建物と一体のデータとして入手。地形だけを Cesium Terrain 用に切り出す場合は変換が必要。            |


### 1.2 配信方法


| 方法                           | 概要                                                                                         | 備考                                                               |
| ---------------------------- | ------------------------------------------------------------------------------------------ | ---------------------------------------------------------------- |
| **Cesium ion 経由**            | ion のアセットとしてストリーミング。アプリ側は `CesiumTerrainProvider` に ion の URL を渡すだけ。                       | 運用不要。ion の利用制限・料金に依存。                                            |
| **自前ホスティング（quantized-mesh）** | DEM 等から quantized-mesh 形式のタイル（`/tiles/{z}/{x}/{y}.terrain`）を生成し、自前サーバーや S3+CloudFront で配信。 | ツール例: cesium-terrain-builder、Cesium ion の地形タイル化パイプライン等。完全な制御が可能。 |
| **サードパーティの Terrain サービス**    | ion 以外で quantized-mesh を配信しているサービスを利用。                                                     | 日本域に特化したサービスは少なく、多くは ion または自前。                                  |

## 2. 地図タイル（影像 — 地形に貼る地図）

### 2.1 入手方法


| 方法                   | 概要                                                                                               | 備考                                     |
| -------------------- | ------------------------------------------------------------------------------------------------ | -------------------------------------- |
| **地理院タイル（国土地理院）**    | 標準地図・空中写真・標高・陰影等、多种のタイルを XYZ 形式で配信。`https://cyberjapandata.gsi.go.jp/xyz/{t}/{z}/{x}/{y}.{ext}`。 | 出典明示で**申請不要**。本プロジェクトのクレジットでも言及。       |
| **MapTiler 等の商用タイル** | 道路・衛星・スタイル付き地図を API キーで利用。日本向けスタイルもある。                                                           | 有料プランあり。キー管理が必要。                       |
| **OpenStreetMap 系**  | OSM データをタイル化したサービス（例: 標準タイル、Mierune 等）。                                                          | クレジット表記が必要。本プロジェクトでは地図に OSM のクレジットを含む。 |
| **自前でタイル生成**         | 地理院タイルや衛星画像をキャッシュし、自前の URL で配信。またはスタイルを変えて再配信。                                                   | 利用規約・出典の確認が必要。                         |


### 2.2 配信方法


| 方法              | 概要                                                       | 備考                                                             |
| --------------- | -------------------------------------------------------- | -------------------------------------------------------------- |
| **公式 URL を直参照** | 地理院タイル等、提供元の URL を `UrlTemplateImageryProvider` にそのまま渡す。 | 実装が簡単。提供元の負荷・規約に依存。                                            |
| **自前キャッシュ・CDN** | タイルを自前の S3 + CloudFront 等にキャッシュし、自前 URL で配信。             | 本プロジェクトでは `MAPTILE_URL` に CloudFront の URL を指定。表示の安定化・規約遵守のため。 |
| **タイルサーバー自前運用** | MapProxy、TileServer GL 等でタイルを配信。                         | 柔軟だが運用コストがかかる。                                                 |

## 3. 建物オープンデータ（3D 建物）

### 3.1 入手方法


| 方法                                 | 概要                                                                                                          | 備考                                                                                                       |
| ---------------------------------- | ----------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| **Cesium ion（Japan 3D Buildings）** | PLATEAU 3D 都市モデル GML を ion 側で 3D Tiles 化したデータ。全国約 2,300 万棟、210 以上の市区町村。テクスチャあり/なし、メタデータあり。無料 ion アカウントで利用可。 | アセット ID 例: 2602291。申請不要・即利用可。                                                                            |
| **PLATEAU（G空間情報センター等）**            | 国土交通省の 3D 都市モデル。CityGML 等の形式でダウンロード可能。対象地域ごとに提供。**提供市区町村**は [G空間情報センターの PLATEAU データセットページ](https://www.geospatial.jp/ckan/dataset/plateau) に都道府県別・市区町村別・整備年（2020〜2025 等）の一覧あり。整備都市の属性リスト（Excel）も同ページから取得可能。Cesium ion の Japan 3D Buildings はこのうち 210 以上の市区町村を 3D Tiles 化して配信。 | 3D Tiles ではないため、Cesium で使うには **変換**（ion の 3D Tiling Pipeline、3d-tiles-tools、CityGML→3D Tiles 変換ツール等）が必要。 |
| **地域限定の 3D Tiles を自前で用意**          | PLATEAU や他ソースを自前で 3D Tiles に変換し、特定地域だけをホスト。                                                                 | 本プロジェクトでは箱根の建物（`bldg_notexture`）を CloudFront で配信している例あり。                                                 |


### 3.2 配信方法


| 方法                     | 概要                                                                                          | 備考                                                  |
| ---------------------- | ------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| **Cesium ion 経由**      | Japan 3D Buildings を ion の 3D Tiles としてストリーミング。`Cesium3DTileset.fromIonAssetId(2602291)` 等。 | 運用不要。ion のストレージ・ストリーミング制限に依存。                       |
| **自前ホスティング（3D Tiles）** | PLATEAU 等を 3D Tiles に変換したうえで、tileset.json とタイルを S3 + CloudFront 等で配信。                       | 本プロジェクトの `TILESET_FEATURES_URL` がこの形態。地域を絞った配信に適する。 |
| **ion にアップロード**        | 自前で変換した 3D Tiles を ion にアップロードし、ion 経由で配信。                                                  | 変換は自前、配信は ion に任せる折衷案。                              |


### 3.3 ion の Japan 3D Buildings でカバーされていない地域での代替方法

Japan 3D Buildings は PLATEAU の 210 以上の市区町村を 3D Tiles 化したものだが、**対象外の市区町村**や**ion に未取り込みの地域**では次のような代替がある。

| 方法 | 概要 | 備考 |
|------|------|------|
| **PLATEAU を自前で取得・変換・配信** | 対象地域が [G空間情報センターの PLATEAU](https://www.geospatial.jp/ckan/dataset/plateau) に含まれていれば、CityGML をダウンロードし、Cesium ion の 3D Tiling Pipeline（KML/COLLADA または CityGML 対応）や 3d-tiles-tools で 3D Tiles に変換。得られた tileset を S3 + CloudFront 等で自前配信する。 | ion の Japan 3D Buildings に「含まれていない」が PLATEAU にはある市区町村を補う場合に有効。本プロジェクトの箱根建物のように地域限定で配信する形態。 |
| **Cesium OSM Buildings（ion）** | Cesium ion が提供する **OpenStreetMap 由来の全球 3D 建物**レイヤー（3 億棟以上）。日本でも OSM に建物フットプリントと高さ等が登録されていれば表示される。 | 品質・解像度は PLATEAU より粗いことが多いが、**PLATEAU 未整備地域**や ion の Japan 3D Buildings 外の場所で、簡易に 3D 建物を出したい場合の代替になる。ion のアセットとして利用可能。 |
| **建物フットプリント＋高さから自前で 3D 化** | OSM（Overpass API 等）や基盤地図情報などから建物ポリゴンと高さを取得し、**押し出し（extrusion）**で簡易 3D メッシュを生成。glTF または 3D Tiles に変換して配信する。 | ツール・パイプラインの開発または既存ツールの組み合わせが必要。細かい形状は出ないが、地域限定で「建物らしい塊」を表示する用途には使える。 |
| **商用・有料の 3D 建物データ** | 特定地域向けの 3D 建物データをベンダーから購入し、3D Tiles 形式に変換（または提供形式のまま）して自前で配信する。 | コストがかかるが、PLATEAU 未整備かつ OSM も粗い地域で品質を確保したい場合の選択肢。 |

※ Japan 3D Buildings と Cesium OSM Buildings を**併用**する場合、同一範囲で両方読み込むと建物が二重になるため、地域に応じてどちらか一方を選ぶか、表示レイヤーを切り替える設計にするとよい。

## 4. 本プロジェクトの現状との対応


| データ種別     | 現状（viewer.js）                                     | 入手                                 | 配信                   |
| --------- | ------------------------------------------------- | ---------------------------------- | -------------------- |
| **3D 地形** | `CesiumTerrainProvider` + ion Asset 770371        | Cesium ion（Japan Regional Terrain） | ion ストリーミング          |
| **地図タイル** | `UrlTemplateImageryProvider` + CloudFront URL     | 国土地理院・OSM 等を元にしたタイル                | CloudFront（自前キャッシュ）  |
| **建物**    | `Cesium3DTileset` + CloudFront の tileset.json（箱根） | PLATEAU 由来を 3D Tiles 化したデータ        | CloudFront（自前ホスティング） |

## 5. 今後の検討の方向

- **地形**: 日本域は ion の Japan Regional Terrain が手軽。自前で範囲・解像度を完全に制御したい場合は、基盤地図・DEM から quantized-mesh を生成して自前配信する選択肢を検討する。
- **地図タイル**: 地理院タイルの直参照 or 自前 CDN キャッシュの使い分け（負荷・規約・表示速度）。商用スタイルが必要なら MapTiler 等を検討する。
- **建物**: 全国をまとめて使うなら ion の Japan 3D Buildings。特定地域だけ・容量や制御を自前にしたい場合は、PLATEAU を 3D Tiles に変換して自前配信する構成を検討する。
