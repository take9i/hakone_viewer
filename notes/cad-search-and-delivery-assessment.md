# CAD データ検索・資材搬入可否判定 — 検討まとめ

本資料では、**(1) CAD データの検索**（敷地形状・周辺道路・建築基準法などの条件での検索、自然言語検索）と **(2) 資材搬入可否の判定**（接道の道路幅・電柱等の障害物の表示・参照、搬入ルート評価）についての検討結果をまとめる。

---

## 第1部：CAD データ検索

### 1.1 要望の整理

- 作成済みの多くの CAD データを、**敷地の形状**・**周辺道路**・**建築基準法を満たす**などの条件で検索したい。
- 検索でヒットした CAD を **Cesium ベースのビューアで即座に表示**できるようにしたい。
- 「この敷地に合った住宅の CAD を検索して」「この家族構成で趣味が○○」のような**自然言語**で検索したい。

### 1.2 既存ソリューションの調査結果

- **敷地形状・周辺道路・建築基準法を条件にした「既存 CAD の検索」に特化した既存アプリ・サービス**は、調査時点では見当たらなかった。
- あるのは次のような「部品」である：
  - **類似図面検索**（CADDi Drawer、図面バンク等）：形状の類似度＋キーワードで図面を検索。敷地・道路・法適合を**メタデータとして付与**すれば、その範囲で条件検索は可能。
  - **敷地・規制のデータ・計算**：基盤地図情報、国交省 API、Map Tools、斜線・セットバック計算ツールなど。検索条件の元データや「適合かどうか」の判定には使えるが、「既存 CAD を条件で検索する」専用サービスではない。

### 1.3 実現方法

検索は次の 2 段階で実現できる。

| 段階 | 内容 |
|------|------|
| **1. メタデータの準備** | CAD に紐づく情報（敷地形状・道路・法規・家族構成・趣味など）を DB に保存する。最初は**手入力**で開始し、必要に応じて AI 抽出や半自動化を追加する。 |
| **2. 検索** | ユーザが**自然言語**で条件を入力し、**Text-to-SQL**（LLM が自然言語を解釈して SQL を生成）で DB を検索し、該当 CAD の一覧を得る。その一覧から選択した CAD を Cesium ビューアで表示する。 |

- **メタデータの付与を軽くする案**：人が**フリーワード**で入力し、**AI がその文から構造化メタデータ（JSON 等）を抽出**して DB に保存する。入力負荷を抑えつつ、検索・Text-to-SQL で使える形に揃えられる。
- **AI によるメタデータ生成**：CAD 図面（PDF／画像等）や立地情報を AI に入力し、敷地形状・接道・用途地域・建ぺい率などのメタデータを自動生成する実例がある（図面認識 AI、VC プロ、不動産図面の AI-OCR 等）。自前で LLM（例：GPT-4 Vision）と組み合わせる構成も可能。

**IFC 形式の場合**：IFC には **Property Set（プロパティセット）・属性・材質**などがファイル内に含まれる。ifcopenshell 等でこれらを抽出し、DB に格納すれば検索用メタデータとしてそのまま利用できる。その DB に対して LLM で自然言語を検索条件（SQL や filterByFormula 等）に変換する構成にすれば、**AI による自然言語検索**も可能である。Cesium ion の Design Tiler で IFC を 3D Tiles にした場合も、メタデータは保持され要素ごとの参照ができる。

### 1.4 規模・難易度の目安

| フェーズ | 内容 | 難易度 |
|----------|------|--------|
| **Phase 1** | メタデータは手動入力のみ。条件検索（敷地形状・道路・法適合等）＋一覧＋Cesium 表示。 | 低〜中 |
| **Phase 2** | メタデータの半自動〜自動（GIS 突合、敷地形状の自動分類、建築基準法の自動計算など）。 | 中〜高 |
| **Phase 3** | 類似形状検索・大規模化・高精度化。 | 高 |

訴求として成立しやすい最小スコープは、CAD 数百件＋手動メタデータ＋条件検索＋Cesium 表示（Phase 1）である。

## 第2部：資材搬入可否判定

### 2.1 要望の整理

- **接している道路の道路幅**と、**電柱などの障害物**の情報を表示する、または参照できるようにする。
- 建設予定地に**資材を運び込めるか**を評価したい。
- データは購入可能との前提。

### 2.2 既存アプリ・サービスの調査結果

- **「建設予定地への資材搬入可否」および「接道の道路幅・電柱などの表示・参照」に特化した既存アプリ・サービス**は、調査時点では見当たらなかった。
- 近いものは次のとおりである：
  - **大型車向けルート検索**（例：トラックカーナビ / NAVITIME）：車高・車幅・全長・重量を入力し、大型車規制（幅員・高さ・重量等）を考慮したルート検索が可能。建設予定地までの「搬入ルート候補」や、その車両で入れそうかの**参考**には使えるが、建設用地専用の「資材搬入可否評価」機能ではない。
  - **建設現場の物流管理**（例：ProperGate）：納品計画・荷下ろし段取り等。搬入可否の事前評価を主目的としたサービスとしては検索では見つからず。
- 搬入可否の**手順**としては、道路幅員・カーブ・電柱・高さ制限・橋梁耐荷重などを**現地で確認**し、必要に応じて走行軌道シミュレーションや特殊車両通行許可の申請を行う、という説明が業界では一般的である。

### 2.3 購入可能なデータ

要望にある「道路幅」「電柱などの障害物」は、以下のようなデータで参照・表示の根拠にできる。

| データ | 提供元 | 主な内容 |
|--------|--------|----------|
| **道路幅員・車両制限** | 日本デジタル道路地図協会（DRM-DB）、国交省 xROAD | 幅員区分、車道幅員、車両制限、橋・トンネル・高架など。建設・土木や特殊車両通行許可の検討に利用される。 |
| **電柱位置** | 東電タウンプランニング等 | 電柱の位置（座標）、電柱番号など。全国の電柱位置データを販売。 |
| **基盤地図情報** | 国土地理院 | 道路縁、建築物外周、街区など（無料）。幅員の直接項目はないが、道路縁から幅員を算出する用途には利用可能。 |

### 2.4 実現の方向性

- **表示・参照**：購入した道路幅員・電柱位置データを **Cesium や地図上に重ねて表示**し、建設予定地周辺の接道・障害物を参照できるようにする。
- **搬入可否の評価**：
  - 車両諸元（幅・高さ・長さ）を入力したうえで、**NAVITIME 等の API** で搬入ルート候補を取得し、自前で「表示した道路幅・電柱との突き合わせ」や簡易ルールによる**搬入可否の評価**を組み合わせる。
  - あるいは、道路・電柱データを**表示・参照用**に用意し、搬入可否はユーザが情報を見て判断する形にする。
- **専用パッケージ**は見つかっていないため、**データを購入し、表示・参照と評価支援を自前アプリで実装する**形が現実的である。
